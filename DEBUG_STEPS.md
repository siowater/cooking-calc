# デバッグ手順

## 現在の状態

最小限のコードに変更しました。`src/App.tsx`をシンプルな画面に変更しています。

## 確認手順

### 1. アプリを再起動

ターミナルで以下を実行：

```bash
# 既存のプロセスを停止（Ctrl+C）
# その後、再度起動
npm start
```

### 2. QRコードをスキャン

- ターミナルに表示されるQRコードをExpo Goアプリでスキャン
- 「Baker's Lens」と「アプリが起動しました！」というメッセージが表示されるはずです

### 3. エラーが表示される場合

#### エラーの種類を確認

**A. Expo Goアプリでエラーが表示される場合**
- エラーメッセージの内容を確認
- ErrorBoundaryがエラーをキャッチしている場合、エラー画面が表示されます

**B. QRコードをスキャンできない場合**
- ターミナルにQRコードが表示されているか確認
- ネットワーク接続を確認（同じWi-Fiに接続されているか）

**C. 「Unable to resolve module」エラー**
- 依存関係が不足している可能性
- `npm install --legacy-peer-deps`を実行

### 4. ログを確認

ターミナルに以下のようなログが表示されるか確認：

```
› Metro waiting on exp://192.168.x.x:8082
› Scan the QR code above with Expo Go
```

### 5. 段階的な機能追加

最小限のコードで動作確認できたら、以下の順序で機能を追加：

1. ✅ 最小限の画面（現在）
2. ナビゲーション追加
3. レシピ一覧画面追加
4. その他の機能追加

## トラブルシューティング

### ターミナルにログが表示されない場合

1. Metro Bundlerが起動しているか確認
2. `npx expo start --clear`でキャッシュをクリア
3. `node_modules`を削除して再インストール

### QRコードが表示されない場合

1. ターミナルウィンドウをスクロールして確認
2. 数秒待ってから再度確認
3. 別のターミナルで`npm start`を実行

### アプリがクラッシュする場合

1. Expo Goアプリでデバイスをシェイク
2. 「Debug Remote JS」を選択
3. Chrome DevToolsでエラーを確認
