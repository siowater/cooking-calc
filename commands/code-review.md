# コードレビュー (Code Review)

未コミットの変更に対する、包括的なセキュリティおよび品質レビュー：

1. 変更されたファイルを取得する: git diff --name-only HEAD
2. 変更された各ファイルについて、以下をチェックする:

**セキュリティ問題 (CRITICAL):**

* ハードコードされた認証情報、APIキー、トークン
* SQLインジェクションの脆弱性
* XSSの脆弱性
* 入力検証の欠落
* 安全でない依存関係
* パストラバーサルのリスク

**コード品質 (HIGH):**

* 50行を超える関数
* 800行を超えるファイル
* 4レベルを超えるネストの深さ
* エラーハンドリングの欠落
* console.log ステートメント
* TODO/FIXME コメント
* パブリックAPIに対するJSDocの欠落

**ベストプラクティス (MEDIUM):**

* 変更（ミューテーション）パターン（代わりに不変パターンを使用する）
* コード/コメントでの絵文字の使用
* 新しいコードに対するテストの欠落
* アクセシビリティの問題 (a11y)

3. 以下を含むレポートを生成する:
* 重大度: CRITICAL, HIGH, MEDIUM, LOW
* ファイルの場所と行番号
* 問題の説明
* 提案される修正


4. CRITICAL または HIGH の問題が見つかった場合、コミットをブロックする

セキュリティ脆弱性のあるコードは決して承認しないでください！
