# テストカバレッジ (Test Coverage)

テストカバレッジを分析し、不足しているテストを生成します：

1. カバレッジ付きでテストを実行する: npm test --coverage または pnpm test --coverage
2. カバレッジレポート (coverage/coverage-summary.json) を分析する
3. カバレッジのしきい値である80%を下回っているファイルを特定する
4. カバレッジ不足の各ファイルについて:
* テストされていないコードパスを分析する
* 関数に対するユニットテストを生成する
* APIに対する統合テストを生成する
* 重要なフローに対するE2Eテストを生成する


5. 新しいテストが通過することを検証する
6. 変更前/変更後のカバレッジメトリクスを表示する
7. プロジェクト全体で80%以上のカバレッジに到達することを保証する

以下に焦点を当てる:

* ハッピーパス（正常系）シナリオ
* エラーハンドリング
* エッジケース (null, undefined, empty)
* 境界条件
