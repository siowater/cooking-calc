
# /learn - 再利用可能なパターンを抽出する

現在のセッションを分析し、スキルとして保存する価値のあるパターンを抽出します。

## トリガー (Trigger)

自明ではない（些細ではない）問題を解決した際、セッション中の任意の時点で `/learn` を実行してください。

## 抽出対象 (What to Extract)

以下を探してください：

1. **エラー解決パターン**
* どのようなエラーが発生したか？
* 根本原因は何だったか？
* 何がそれを修正したか？
* これは同様のエラーに対して再利用可能か？


2. **デバッグ技術**
* 自明ではないデバッグ手順
* 効果的だったツールの組み合わせ
* 診断パターン


3. **回避策 (Workarounds)**
* ライブラリの癖（仕様上の特異な挙動）
* APIの制限
* バージョン固有の修正


4. **プロジェクト固有のパターン**
* 発見されたコードベースの規約
* 行われたアーキテクチャ決定
* 統合パターン



## 出力フォーマット (Output Format)

`~/.cursor/skills/learned/[pattern-name].md` にスキルファイルを作成します：

```markdown
# [記述的なパターン名]

**Extracted:** [日付]
**Context:** [これが適用される場面の簡単な説明]

## Problem
[これが解決する問題 - 具体的に]

## Solution
[パターン/テクニック/回避策]

## Example
[該当する場合、コード例]

## When to Use
[トリガー条件 - 何がこのスキルを有効化すべきか]

```

## プロセス (Process)

1. 抽出可能なパターンがないかセッションをレビューする
2. 最も価値があり再利用可能な洞察を特定する
3. スキルファイルを起草する
4. 保存する前にユーザーに確認を求める
5. `~/.cursor/skills/learned/` に保存する

## 注意事項 (Notes)

* 些細な修正（タイプミス、単純な構文エラー）は抽出しないでください
* 一回限りの問題（特定のAPI停止など）は抽出しないでください
* 将来のセッションで時間を節約できるパターンに焦点を当ててください
* スキルは焦点を絞ってください - 1スキルにつき1パターン
