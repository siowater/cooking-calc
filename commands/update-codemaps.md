# コードマップの更新 (Update Codemaps)

コードベースの構造を分析し、アーキテクチャドキュメントを更新します：

1. インポート、エクスポート、および依存関係について、すべてのソースファイルをスキャンする
2. 以下の形式で、トークン効率の良い（token-lean）コードマップを生成する：
* codemaps/architecture.md - 全体的なアーキテクチャ
* codemaps/backend.md - バックエンド構造
* codemaps/frontend.md - フロントエンド構造
* codemaps/data.md - データモデルとスキーマ


3. 以前のバージョンからの差分の割合（パーセンテージ）を計算する
4. 変更が30%を超える場合、更新する前にユーザーの承認を求める
5. 各コードマップに鮮度のタイムスタンプ（更新日時）を追加する
6. レポートを `.reports/codemap-diff.txt` に保存する

分析にはTypeScript/Node.jsを使用してください。実装の詳細ではなく、ハイレベルな構造に焦点を当ててください。
