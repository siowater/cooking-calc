# ドキュメントの更新 (Update Documentation)

信頼できる情報源（ソース・オブ・トゥルース）からドキュメントを同期します：

1. package.json の scripts セクションを読み込む
* スクリプトリファレンス表を生成する
* コメントからの説明を含める


2. .env.example を読み込む
* すべての環境変数を抽出する
* 目的とフォーマットを文書化する


3. 以下を含む docs/CONTRIB.md を生成する：
* 開発ワークフロー
* 利用可能なスクリプト
* 環境セットアップ
* テスト手順


4. 以下を含む docs/RUNBOOK.md を生成する：
* デプロイ手順
* 監視とアラート
* 一般的な問題と修正
* ロールバック手順


5. 古くなったドキュメントを特定する：
* 90日以上変更されていないドキュメントを見つける
* 手動レビューのためにリストアップする


6. 差分サマリーを表示する

信頼できる唯一の情報源：package.json および .env.example
