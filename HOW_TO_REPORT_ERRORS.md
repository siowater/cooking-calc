# エラーメッセージの共有方法

## 方法1: スクリーンショットを共有（最も簡単）

1. **iPhoneでエラー画面のスクリーンショットを撮る**
   - iPhone X以降: サイドボタンと音量ボタンを同時に押す
   - iPhone 8以前: ホームボタンとサイドボタンを同時に押す

2. **スクリーンショットをCursorに貼り付け**
   - Cursorのチャット画面に画像をドラッグ&ドロップ
   - または、画像ファイルを添付

## 方法2: エラーメッセージをコピー

### iPhoneでエラーメッセージをコピーする方法

1. **エラー画面を長押し**
   - エラーメッセージのテキストを長押し
   - 「すべて選択」をタップ
   - 「コピー」をタップ

2. **Cursorに貼り付け**
   - コピーしたテキストをCursorのチャット画面に貼り付け

### エラーメッセージが長い場合

1. **エラー画面をスクロール**
   - エラーメッセージ全体をスクロールして確認
   - 特に重要な部分（エラーの種類、ファイル名、行番号）を探す

2. **重要な部分をコピー**
   - エラーの種類（例: "Error:", "TypeError:", "Cannot read property"）
   - ファイル名と行番号（例: "App.tsx:15"）
   - エラーメッセージの最初の数行

## 方法3: ターミナルのログを確認

ターミナルにエラーメッセージが表示されている可能性があります。

### ターミナルでエラーログを確認

1. **`npm start`を実行しているターミナルを確認**
   - 赤い文字で表示されているエラーメッセージを探す
   - `ERROR`で始まる行を探す

2. **エラーメッセージをコピー**
   - ターミナルでエラーメッセージを選択
   - コピー（Ctrl+C または右クリック→コピー）
   - Cursorに貼り付け

## 方法4: Expo Goアプリのデバッグモード

### デバッグモードを有効化

1. **iPhoneをシェイク（振る）**
2. **「Debug Remote JS」を選択**
3. **Chrome DevToolsが開く**
4. **Consoleタブでエラーメッセージを確認**
5. **エラーメッセージをコピーしてCursorに貼り付け**

## 方法5: エラーメッセージの一部を共有

エラーメッセージが長すぎる場合、以下の情報だけでも共有してください：

1. **エラーの種類**
   - 例: "TypeError", "ReferenceError", "Cannot read property"

2. **ファイル名と行番号**
   - 例: "App.tsx:15", "RecipeListScreen.tsx:42"

3. **エラーメッセージの最初の1-2行**
   - 例: "Cannot read property 'navigation' of undefined"

4. **エラーが発生した操作**
   - 例: "レシピ一覧画面を開いた時", "「+」ボタンをタップした時"

## 推奨される方法

**最も簡単で確実な方法**:
1. **スクリーンショットを撮る**
2. **Cursorに画像を貼り付け**

これで、エラーメッセージ全体を正確に共有できます。

## よくあるエラーパターン

以下のようなエラーが表示される場合、その情報だけでも共有してください：

- `Cannot read property 'navigation' of undefined`
- `TypeError: undefined is not an object`
- `Error: Element type is invalid`
- `Unable to resolve module`

これらのエラーメッセージの最初の部分だけでも、問題を特定できることがあります。
