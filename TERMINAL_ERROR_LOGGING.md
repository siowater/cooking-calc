# ターミナルエラーログ機能

## ✅ 実装完了

エラーが発生した場合、**iPhone側のエラーと同じ内容がターミナルにも表示される**ようになりました。

## 実装内容

### 1. ErrorBoundaryの強化

`ErrorBoundary`コンポーネントがエラーをキャッチした際に、ターミナルに詳細な情報を出力します：

```
═══════════════════════════════════════════════════════
🚨 ERROR BOUNDARY: エラーが発生しました
═══════════════════════════════════════════════════════
エラーメッセージ: [エラーメッセージ]
エラーオブジェクト: [エラーオブジェクト]
スタックトレース: [スタックトレース]
コンポーネントスタック: [コンポーネントスタック]
エラー情報: [エラー情報]
═══════════════════════════════════════════════════════
```

### 2. グローバルエラーハンドラー

未処理のエラーもターミナルに出力されます：

```
═══════════════════════════════════════════════════════
🚨 GLOBAL ERROR HANDLER: 未処理のエラーが発生しました
═══════════════════════════════════════════════════════
エラーメッセージ: [エラーメッセージ]
エラーオブジェクト: [エラーオブジェクト]
スタックトレース: [スタックトレース]
致命的なエラー: [はい/いいえ]
═══════════════════════════════════════════════════════
```

### 3. Promise Rejection ハンドラー

未処理のPromise拒否もターミナルに出力されます：

```
═══════════════════════════════════════════════════════
🚨 UNHANDLED PROMISE REJECTION: Promiseが拒否されました
═══════════════════════════════════════════════════════
拒否された理由: [理由]
エラーオブジェクト: [エラーオブジェクト]
スタックトレース: [スタックトレース]
═══════════════════════════════════════════════════════
```

### 4. 各画面のエラーハンドリング

各画面のエラーハンドリングでも、ターミナルに詳細なログを出力します：

- `RecipeListScreen`: レシピの読み込み、削除、お気に入り更新のエラー
- その他の画面も同様に実装予定

## 使い方

### エラーが発生した場合

1. **iPhone側でエラーが表示される**
2. **同時にターミナルにも同じエラーが表示される**
3. **ターミナルのエラーメッセージをコピーして共有**

### ターミナルでエラーを確認

`npm start`を実行しているターミナルウィンドウで、以下のような形式でエラーが表示されます：

```
═══════════════════════════════════════════════════════
🚨 ERROR BOUNDARY: エラーが発生しました
═══════════════════════════════════════════════════════
...
```

## メリット

1. **デバッグが簡単**: iPhone側のエラーと同じ内容がターミナルにも表示される
2. **エラーメッセージの共有が容易**: ターミナルからコピー&ペーストできる
3. **詳細な情報**: スタックトレースやコンポーネントスタックも表示される
4. **未処理エラーも捕捉**: グローバルエラーハンドラーで未処理のエラーも捕捉

## 注意事項

- エラーログは開発環境でのみ表示されます
- 本番環境では、エラーログは表示されません（セキュリティ上の理由）
- ターミナルに表示されるエラーログは、Metro Bundlerが実行されている場合のみ表示されます
