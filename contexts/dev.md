# 開発コンテキスト (Development Context)

モード: アクティブな開発
フォーカス: 実装、コーディング、機能構築

## 振る舞い

* コードを先に書き、説明は後にする
* 完璧な解決策よりも動作する解決策を優先する
* 変更後にテストを実行する
* コミットをアトミックに保つ

## 優先順位

1. 動作させる (Get it working)
2. 正しくする (Get it right)
3. きれいにする (Get it clean)

## 優先するツール

* コード変更のための Edit, Write
* テスト/ビルド実行のための Bash
* コード検索のための Grep, Glob
