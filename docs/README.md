# Baker's Lens ドキュメント一覧

このディレクトリには、Baker's Lensアプリの開発に必要なドキュメントが含まれています。

## 📚 ドキュメント構成

### 1. [要件定義書](./requirements.md)
プロジェクトの概要、機能要件、非機能要件を定義したドキュメントです。

**内容**:
- プロジェクト概要
- ターゲット・課題
- 機能要件（OCR、スケーリング、計量支援など）
- 技術スタック
- 非機能要件

### 2. [アーキテクチャ設計書](./architecture.md)
システム全体のアーキテクチャ設計を記載したドキュメントです。

**内容**:
- システムアーキテクチャ図
- コンポーネント構成
- データフロー
- 技術スタック詳細
- パフォーマンス要件
- セキュリティ設計
- アーキテクチャ決定記録（ADR）

### 3. [データベース設計書](./database-design.md)

**内容**:

### 4. [API設計書](./api-design.md)

**内容**:

- ストレージAPI
- ユーザー設定API

### 5. [UI/UX設計書](./ui-ux-design.md)
ユーザーインターフェースとユーザーエクスペリエンスの設計を記載したドキュメントです。

**内容**:

### 6. [技術仕様書](./technical-specification.md)

## 🚀 クイックスタート

### 新規開発者の場合

1. [要件定義書](./requirements.md)を読んでプロジェクトの概要を理解する
2. [開発環境セットアップガイド](./setup-guide.md)に従って環境を構築する
3. [アーキテクチャ設計書](./architecture.md)でシステム構成を理解する
4. [技術仕様書](./technical-specification.md)で実装の詳細を確認する

### 機能開発の場合

1. [要件定義書](./requirements.md)で機能要件を確認
2. [アーキテクチャ設計書](./architecture.md)で影響範囲を確認
3. [データベース設計書](./database-design.md)でデータモデルを確認（必要に応じて）
4. [API設計書](./api-design.md)でAPI仕様を確認（必要に応じて）
5. [UI/UX設計書](./ui-ux-design.md)でUI設計を確認（必要に応じて）
6. [技術仕様書](./technical-specification.md)で実装の詳細を確認
7. [テスト計画書](./test-plan.md)に従ってテストを作成

### デプロイメントの場合

1. [デプロイメントガイド](./deployment-guide.md)を参照
2. リリース前チェックリストを確認
3. ビルドと配布を実行

## 📖 ドキュメントの更新

ドキュメントは、プロジェクトの進捗に合わせて更新してください。

- アーキテクチャ変更時: [アーキテクチャ設計書](./architecture.md)を更新
- データベース変更時: [データベース設計書](./database-design.md)を更新
- API変更時: [API設計書](./api-design.md)を更新
- UI変更時: [UI/UX設計書](./ui-ux-design.md)を更新
- 技術仕様変更時: [技術仕様書](./technical-specification.md)を更新

## 📝 ドキュメント作成のベストプラクティス

1. **明確性**: 専門用語を使う場合は説明を追加
2. **具体性**: コード例や図を積極的に使用
3. **最新性**: コード変更に合わせてドキュメントを更新
4. **検索性**: 適切な見出しとインデックスを使用
5. **日本語**: すべてのドキュメントは日本語で記述
